   另外，三者拼在一起，为混合背包
2. 从决策角度，01背包 -> 分组背包 -> 有依赖背包(用树形DP求解)

4. 其他扩展：求方案数、具体方案



#### 特点总结
1. 所有背包问题中，只有**完全背包**在优化后，体积是从小到大的(即`f[i][j-x]`型，其他都是`i-1`)
2. 代码顺序：**for物品->for体积->for决策**。其中物品和体积是状态表示的两层(必然枚举)，决策是根据状态转移方程来的
   **所谓决策，其实就是状态转移方程的各个分段函数**
   如01背包，决策只有两个；如完全背包，每个物品选任意个，决策就一直枚举到体积限制；如LIS问题，决策j枚举到i-1
   并不是所有的决策都依赖循环，像01背包，决策项只有两个，没有写成循环的必要



#### 状态表示总结
1. 从前i个物品中选，花费**最多**是j的选法**（最常见）**
2. 从前i个物品中选，花费**恰好**是j的选法
3. 从前i个物品中选，花费**最少**是j的选法

这三种模型，在状态计算上，简单划分为两个子状态：**选i**和**不选i**

**不选i：**`f[i-1][j]`

**选i：**这一点上，三种表示方法产生了分歧

形式上，都是`f[i-1][j-v]`

条件上，**1**和**2**都要求`j-v>=0`，而**3**则没有这个要求

当然，数组下标不允许是负数；不过对于**3**来说，j为负数时的状态，与j为0没有区别（至少花费负数和至少花费0，都可以归为至少花费0）

相比**1**和**2**，如果**3**遇到`j-v<0`的情况，状态转移方程也应当涉及到`f[i-1][j-v]`，写作`f[i-1][0]`，而非直接不合法



#### DP初始化问题
依然以上述三个状态表示为分类依据进行总结

注意，状态表示的属性值，可能是价值(属性为max/min)，可能是方案数(属性为cnt)；这里均会给出

只有两种特殊情况，分别出现在**2**和**3**中，且都为`f[0][j]`

1. 至多模型：
   `f[0][j] = 0/1`:没有物品可选，花费最多是j的价值为0 / 方案数为1，即什么都不选
   `f[i][0] = 0/1`:花费最多是0的选法，价值为0 / 方案数为1——什么都不选
2. 恰好模型（根据j是否为0有区别）：
   `f[0][j] = -1/0`:没有物品可选，花费恰好是j的选法不存在 / 方案数为0
   `f[0][0] = 0/1`:没有物品可选，花费恰好是0的选法的价值为0 / 方案数为1，即什么都不选
   `f[i][0] = 0/1`:花费恰好是0的选法，价值为0 / 方案数为1——什么都不选
3. 至少模型（根据j是否为0有区别）：
   `f[0][j] = -1/0`:没有物品可选，花费至少是j的选法不存在 / 方案数为0，即不合法
   `f[0][0] = 0/1`:没有物品可选，花费至少是0的选法的价值为0 / 方案数为1，即什么都不选
   `f[i][0] = 0/1`:花费至少是0的选法，价值为0 / 方案数为1——什么都不选

再提一点，对于不合法情况的初始化，应根据属性是max/min，选择初始化为负无穷/正无穷



#### DP循环起点问题
以**for物品->for体积->for决策**为例：
1. 物品循环起点：`i=1`，因为`f[0][j]`一般根据定义来初始化，以处理`i-1`层的越界
2. 体积循环起点：`j=0`，花费是0的选法一般是合法的
3. 决策循环起点：不定，根据状态转移方程的分段情况来定
