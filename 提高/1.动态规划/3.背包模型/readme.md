#### 特点总结
1. 所有背包问题中，只有**完全背包**在优化后，体积是从小到大的(即`f[i][j-x]`型，其他都是`i-1`)
2. 代码顺序：**for物品->for体积->for决策**。其中物品和体积是状态表示的两层(必然枚举)，决策是根据状态转移方程来的
   **所谓决策，其实就是状态转移方程的各个分段函数**
   如01背包，决策只有两个；如完全背包，每个物品选任意个，决策就一直枚举到体积限制；如LIS问题，决策j枚举到i-1
   并不是所有的决策都依赖循环，像01背包，决策项只有两个，没有写成循环的必要



#### 状态表示总结
1. 从前i个物品中选，花费**最多**是j的选法**（最常见）**
2. 从前i个物品中选，花费**恰好**是j的选法
3. 从前i个物品中选，花费**最少**是j的选法

这三种模型，在状态计算上，简单划分为两个子状态：**选i**和**不选i**

**不选i：**`f[i-1][j]`

**选i：**这一点上，三种表示方法产生了分歧

形式上，都是`f[i-1][j-v]`

条件上，**1**和**2**都要求`j-v>=0`，而**3**则没有这个要求

当然，数组下标不允许是负数；不过对于**3**来说，j为负数时的状态，与j为0没有区别（至少花费负数和至少花费0，都可以归为至少花费0）

相比**1**和**2**，如果**3**遇到`j-v<0`的情况，状态转移方程也应当涉及到`f[i-1][j-v]`，写作`f[i-1][0]`，而非直接不合法



#### DP初始化问题
依然以上述三个状态表示为分类依据进行总结

首先明确，初始化部分只关注f[0][j]的情况；对于滚动数组来说，恰好就是f数组当前所在的第0层

注意，状态表示的属性值，可能是价值(属性为max/min)，可能是方案数(属性为cnt)；这里均会给出

一般来说，cnt的初始化方法是在max/min基础上+1

又因为全局数组默认初始化为0，对于属性为max/min的题目来说，f数组很多时候不需要初始化；

其中，至多模型j不区分0；恰好和至少模型，j区分0

1. 至多模型：
   `f[0][j] = 0/1`:没有物品可选，花费最多是j的价值为0 / 方案数为1，即什么都不选
2. 恰好模型（根据j是否为0有区别）：
   `f[0][j] = -1/0`:没有物品可选，花费恰好是j的选法不存在 / 方案数为0
   `f[0][0] = 0/1`:没有物品可选，花费恰好是0的选法的价值为0 / 方案数为1，即什么都不选
3. 至少模型（根据j是否为0有区别）：
   `f[0][j] = -1/0`:没有物品可选，花费至少是j的选法不存在 / 方案数为0，即不合法
   `f[0][0] = 0/1`:没有物品可选，花费至少是0的选法的价值为0 / 方案数为1，即什么都不选

再提一点，对于不合法情况-1的初始化，应根据属性是max/min，选择初始化为负无穷/正无穷

最后，对上述内容进行简单总结；对于滚动数组，自动把`f[0][j]`视为`f[j]`即可
1. 如果是至多模型：
   属性为`max/min`，`f`自然初始化为`0`即可
   属性为`cnt`，初始化`f[0][0~j]`为`1`
2. 如果是恰好模型和至少模型：
3. 属性为`max/min`，`f[0][0]`初始化为`0`，`f[0][1~j]`初始化为`-∞/+∞`
   属性为`cnt`，`f[0][0]`初始化为`1`，`f[0][1~j]`初始化为`0`



#### DP循环起点问题
以**for物品->for体积->for决策**为例：
1. 物品循环起点：`i=1`，因为`f[0][j]`一般根据定义来初始化，以处理`i-1`层的越界
2. 体积循环起点：`j=0`，花费是0的选法一般是合法的
3. 决策循环起点：不定，根据状态转移方程的分段情况来定



#### DP与最短路问题的关系
见10.混合背包问题
先下结论：DP = 有向无环图(DAG) + 拓扑序
对于状态数组f[i][j]，(i,j)视为图中的点；这里可以用二维坐标理解
图中共n*m个点，每个点(i,j)表示前i个物品中选，体积不超过j的集合
for物品for体积 -> 遍历图中所有点
for决策+状态转移方程 -> 遍历所有入边并决定从哪条转移而来 -> 松弛操作
提示：for决策就是用来表述复杂的状态转移方程的
属性为max/min/cnt影响状态转移方程的写法，进而影响从哪条入边转移而来，从而影响整条路径

在路径推进上，最短路问题和DP的区别是什么呢？
1. 寻路问题
Dijkstra：处理的是可能有环但边权非负的图，通过“贪心+优先队列”来动态寻找拓扑序。
DP：因为结构太规律了，总是从i-1传到i，不需要优先队列去维护，直接用嵌套循环就能走出完美的拓扑序。
DP的这种层序视角——算完(i-1,x)这一层的点，才能算(i,x)这一层——其实理解为拓扑序更恰当
1. 松弛操作
Dijkstra：通过松弛操作，在当前点遍历出边，来更新出边点的最短距离，从而对下一个出队对象产生影响，这直接影响到路径构建
DP：通过状态转移方程，在当前点遍历入边，来确定要选择哪个起点来构建路径
两种松弛操作，一个是对出边进行松弛(Push)，一个是对入边进行松弛(Pull)

以01背包：f(i,j)=max{ f(i-1,j), f(i-1,j-v[i])+w[i] }为例
计算DP的过程和遍历图一样，一层层向外扩散，先得到距离起点近的(i-1,j)的状态，再得到距离起点更远的(i,j)的状态
通过式子可以发现，(i,j)点有两条入边，分别来自(i-1,j)和(i-1,j-v[i])两个点；根据状态转移方程，来决定要从哪条入边转移而来

for物品for体积for决策，相当于遍历图中的所有点和边，在图论中是O(V+E)的复杂度
而最短路的特征是什么？Dijikstra是贪心的，遍历到点n时，该点一定确定了最短路
获取图中所有路径是指数级别的，那太恐怖；但最短路算法，只需要层序遍历图，就可以不断地把差方案筛掉，从结果上来看，就是走到该点时，最短路已确定
前者是DFS，后者是BFS；这也就是图论部分讲到的，BFS相比DFS天生具备维护“最优”的性质
这时，惊奇地发现：图论中层序的推进，不就是DP“无后效性”的生动体现吗？
DP的无后效性，指的是状态转移方程中，状态的计算只依赖于前面已经计算过的状态；图论中层序的推进，也要求先处理完上一层的点，才能处理下一层的点
