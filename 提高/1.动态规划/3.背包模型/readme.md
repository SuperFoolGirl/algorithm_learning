#### 变种问题
> 01背包作为母题
1. 01背包 -> 完全背包 -> 多重背包，三者是递进的
   另外，三者拼在一起，为混合背包
2. 从决策角度，01背包 -> 分组背包 -> 有依赖背包(用树形DP求解)
3. 维度扩展：二维费用问题
4. 其他扩展：求方案数、具体方案
#### 特点总结
1. 所有背包问题中，只有**完全背包**在优化后，体积是从小到大的(即`f[i][j-x]`型，其他都是`i-1`)
2. 代码顺序：for物品->for体积->for决策。其中物品和体积是状态表示的两层(必然枚举)，决策是根据状态转移方程来的(依赖代码循环的方程)
   如完全背包，每个物品选任意个，决策就一直枚举到体积限制；如LIS问题，决策j枚举到i-1
