### 定义
首先，强连通分量(SCC)是针对有向图而言的

对于一个有向图，**连通分量**：对于分量中任意点u,v，必然可以从u走到v，也可从v走到u，**即任意两点之间可相互到达**

**强连通分量**：**极大**连通分量——如果一个连通分量加上任意一个点就不再是连通分量了，那么这个连通分量就是强连通分量

最小的SCC是**孤立点**，无环

含有**多个顶点**的SCC必然**至少存在一个环**。由于各点需要互相可达，因此这个连通块是由环构成的，任何一个点至少属于一个环的一部分；可能存在一些点同时属于多个环

由于SCC的极大性，因此不存在一个点同时属于两个SCC，这种情况下，两个SCC就可以合并成一个更大的SCC了



### 作用
将一个有向图通过**缩点**的方式，转化为有向无环图(拓扑图DAG)。

注意，**DAG不要求连通**。若原图不连通，求出的DAG也一定不连通。但是**不影响拓扑排序**，拓扑排序一开始将所有入度为0的点入队，然后同时跑BFS；**可以把拓扑排序看作“DAG上所有连通块同步扩展”**

**缩点：将所有强连通分量缩成一个点**。这个思路，在学最短路时，将连通块视为一个点的做法很相似

给定一张有向图，将图中所有的强连通分量缩成一个点，就能得到拓扑图

基于偏序关系的DAG，可以用递推的方式以线性时间复杂度求最短路



### 前置知识
用dfs，搜索方式为先序遍历。可以用有向树的视角，以便引入“高度”，搜索时将**边**分为四类：

1. 树枝边：普通的父子边，即x->y
2. 前向边：x是y的祖宗节点，有一条x指向y的边，且x与y之间至少间隔一个节点
3. 后向边：x是y的祖宗节点，有一条y指向x的边，且x与y之间至少间隔一个节点
4. 横插边：当前点的出边指向左边的点(即已访问过的点，tarjan-lca中规定的2类的点)；注意往右的边(指向0类点)不属于横插边，而是普通的树枝边

如果一个点x在SCC中，有以下情况：
1. 从**后向边**(指向自己的一个祖先节点)走到当前正在搜索的路径上(当前搜索的路径即为tarjan-lca中规定的w链)，进而又能回到x
2. 通过**横插边**拜访已经访问过的2类点，然后由横插边走到x的某个祖先，进而又回到x

多节点SCC必然存在环，而只有后向边和横插边才可能形成环；前向边不存在形成环的可能



### Tarjan算法
对每个点定义两个时间戳

**dfn[u]：第一次遍历到u的时间戳(按照先序遍历)**，depth-first number，深度优先序

**low[u]：从u出发一直走下去，所能遍历到的最小dfn是什么**

**对于SCC上深度最小(高度最大)的点u，一定有dfn[u] = low[u]**

解释：从u开始走的话，无论如何都走不到u上面(高度更高)的点，即走不到dfn更小的点；遍历到u时记录下的dfn[u]，而从u出发一定能走到u，根据规则，走不到更小的dfn的点，因此low[u]也只能是dfn[u]

刚才上面讲到，对于深度大的点x，可能通过横插边经过2类点y回到u进而回到x；但根据先序遍历，dfn[y]也一定大于dfn[u]

根据上面描述的过程，**整个SCC上的点的low值都等于dfn[u]**，因此就能通过dfn和low的关系来判断SCC的边界了

##### 代码模板
时间复杂度为$O(n+m)$，每个点都有一个递归函数，同时每个点都要遍历其出边
```c++
// 注意，树不是SCC，这里只是借用了“树的视角”，目的是借助树独有的深度、先序遍历方式来简化理解
// 例如提到“叶子节点”，其实指的是原图中出度为0的点；深度也可以理解为拓扑序列中的位置
void tarjan(int u) {
    // 访问u，记录dfn和low为时间戳
    // 1.对于属于的SCC"非最高点"，由于总能走到最高点，low会在递归时更新为最高点的dfn
    // 2."叶子节点"的low就是dfn，因为它没有儿子了，无法更新low了
    // 3.对2进行扩展：对于图中所有不属于SCC的点，都满足low=dfn(即使有路可走，但往下走的话，可达点的dfn一定严格更大)，这些点都各自构成一个独立的SCC
    dfn[u] = low[u] = ++timestamp;

    // stk维护当前SCC的点，处于正在搜索的过程中
    // 并非dfs的调用栈(只维护1类点)
    // in_stk标记当前点是否在stk中
    stk[++top] = u;
    in_stk[u] = true;

    for (int i = h[u]; ~i; i = ne[i]) {
        int j = e[i];
        if (!dfn[j]) {
            // j还未遍历到，为0类点
            tarjan(j);
            // 此时j已经递归结束，已经求出了low[j]真实值(确信递归能完成它的任务)
            // j能到的最小值，u也能到；用儿子们来更新u
            low[u] = min(low[u], low[j]);
        } else if (in_stk[j]) {
            // j已遍历过，且已入栈，说明是u->j是后向边或横插边
            // 这种情况下，应当用dfn[j]，而不是low[j]
            // 因为u->j不是树枝边，只有树枝边才能维护“父子关系”，父亲才能继承儿子的low

            // 这里也体现出in_stk的作用了：
            // 此前，可能已经有非SCC节点被遍历并处理结束了，它们的dfn不为0，但已经不在stk中了
            // 这些点已经完全不再需要任何考虑了
            // 因此这里不能直接写"else"
            low[u] = min(low[u], dfn[j]);
        }
    }

    // stk中从栈顶到u的部分全都是属于同一个SCC的，可以画图模拟
    // 根据定义，"叶子节点"一定是SCC
    // 因此"叶子节点"执行至此会直接出栈，不会影响其他栈中其他SCC的划分
    // 从这里可以看出来，跑一遍tarjan可以找出所有的SCC
    if (dfn[u] == low[u]) {
        scc_cnt++;
        int y;
        // do-while: 第一次无条件执行，满足while后继续执行，直到执行后不满足while
        // do-while的合理性：
        // 出栈至少执行一次，对应SCC至少由一个点构成
        // 且出栈元素存储在y中，可以直接供while检测本轮出栈元素——若已经出栈的y为SCC最高点，就停止出栈
        do {
            // 取出栈顶元素
            y = stk[top--];
            in_stk[y] = false;
            // 将y归入当前SCC
            id[y] = scc_cnt;
        } while (y != u);
    }
}
```



### 后续处理
#### 缩点
经过上述处理，所有SCC的信息都存储在了id中，id[i]表示点i所属的SCC编号

在原图的基础上，"建一个新图"。**新图中点的个数，就是SCC的个数，因此新点的编号就是SCC的编号(与下方提到的拓扑序有关)**

"建新图"之所以打引号，是因为题目不一定需要真的建出一个图来。由于新图是DAG，通过统计各点入度、出度就可以获取新图信息了，无需建图增大开销。一般，**出度用于求终点(dout=0)，入度用于求起点(din=0)**

注意，原图中所有无法构成SCC的点，它们每一个都作为独立的SCC

遍历所有边，方式：遍历所有点，然后遍历当前点i的出边，找到邻点j

如果i和j不在同一个SCC中(即id[i] != id[j])，则在缩点后的图中添加一条从id[i]指向id[j]的边

否则，无需处理

这样遍历完所有的边后，新图呈现的状态就是，在原图的基础上，所有SCC都被缩成了一个点，得到了DAG

**另外，SCC编号(从1开始)递减的顺序(scc_cnt从大到小)就是DAG的拓扑序了**，可以自行模拟验证(原理是DFS求拓扑排序，这里不多涉及；一般拓扑排序都是BFS求解的)；因此，不需要再进行一次拓扑排序了
