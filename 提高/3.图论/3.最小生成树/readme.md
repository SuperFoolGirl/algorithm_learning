kruskal的泛用性比prim更强；能用prim的图也能用kruskal，但反过来不一定

#### prim

prim算法：维护一棵树，每次从外界选择一条权值最小的边加入树中，保证加边后仍然为树，直到树包含所有顶点为止

时间复杂度：$O(n^2)$，用邻接矩阵存图

证明：当前与外界直接相连的权值最小的一条边ab，这条边一定可以出现在最优解中；此时a在树中，b在树外

反证：此时ab不连通，由于所有点都会被加入到mst，若不选该边，后续ab会以其他路径x连通。视边ab与“其他路径x”一起形成环，x上有若干边，其中至少有一条边y权值>=ab的权值，因为它的出现和选择在ab之后，ab先于y被选中，说明ab边权更小；由于是无向边，因此y若用ab来替代，则会得到更好的结果；因此矛盾

#### kruskal

kruskal算法：每次从小到大选择一条边加入树中，保证加边后仍然为树，直到树包含所有顶点为止

特点：处理所有排序好的边时，**假设当前已经处理完第i条边，说明已经求出了由前i条边构成的所有连通块**；那么，当处理完所有边时，说明求出了由所有点构成的连通块，即最小生成树
可以看到，**“求当前连通块个数”是在求mst的过程中自然产生的一个副产品**
*扩展应用-第二题处应用了这个特点*

时间复杂度：$O(m\log m)$，用三元组存边

证明：当前权值最小的一条边ab，这条边一定可以出现在最优解中

反证：思路类似同上：此时ab不连通，但ab总会连通；如果不选该边，后续ab会以其他路径x连通。视边ab与“其他路径x”一起形成环，x上有若干边，其中至少有一条边y权值>=ab的权值，因为它的出现和选择在ab之后，排序说明y边权更大；由于是无向边，因此y若用ab来替代，则会得到更好的结果；因此矛盾

#### 思想应用
如何证明当前边a一定可以选？

反证：假设不选当前边，得到了一棵树；将a加上后，那么必然出现环；在这个环上，一定可以找到一条长度不小于a的边，那么用a来替换这条边，结果一定不会变差

#### 最小生成树的深入理解
kruskal和prim都是基于这部分理解的，详见第四题对于kruskal的剖析。这里做个总结：

1. 任意一棵最小生成树一定可以包含无向图中权值最小的边(证明：反证出环)
2. 设无向图共有n个点m条边，其中k条已经选出，构成了生成森林(一系列连通块)；可以将这些连通块视为“点”，再从剩余m-k条边中选择n-k-1条加入生成森林，使之成为原图的最小生成树，则连接某两连通块的边一定包含能够连通这两块的、权值最小的边
