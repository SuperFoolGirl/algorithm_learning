|              |          BFS           |           Dijkstra           |        SPFA        |
| :----------: | :--------------------: | :--------------------------: | :----------------: |
|  时间复杂度  |        $O(n+m)$        |         $O(m\log n)$         | $O(km),最坏O(nm)$  |
|   队列限制   | 只能进队一次，出队一次 | 可多次进队，只可有效出队一次 |   可多次进队出队   |

|     备注     |      用于无边权图      |   0-1BFS属于Dijkstra的一种   |   可用于负权边图   |

### 拓扑图

1. 求负权边只有两种方式：SPFA或拓扑排序（只能用于拓扑图）

  其中，按照拓扑序求最短路，本质上就是“关键路径求最短路”，都是线性复杂度

2. BFS和Dijkstra的扩展过程天然满足拓扑序，能够保证递推的正确性，进而保证最短路算法的正确性

	而SPFA并不是同一路数：其本质已经说过，让所有边打擂台，而并不依赖一次性精准递推



### 最短路拓扑图

在非负权图上，为了记录具体方案，常用pre数组辅助——pre[i]表示转移到i的点x，转移路径x->i

该方法只使用于BFS和Dijkstra，分别用于无权图/非负权图

称之为“最短路拓扑图”，因为pre数组还原出来的路径一定满足拓扑序

拓扑序还意味着“最短路的更新顺序”，以Dijkstra为例，出队即代表“最短路确定”，也就是更新结束

注意，并没有讨论原图的拓扑性，原图有无向尚且未知，这里只讨论“最短路”构成的拓扑图

**满足拓扑序的意义是：确保集合在计算时，其子集能够全部被算出来，进而能够正确地计算出集合**

**常常用于统计其他“依赖递推关系”的属性，如统计最短路数量**

**“最短路更新”、“最短路数量统计”是同步的；因此，只有最短路确定满足拓扑序，才能构建正确的递推关系，最短路数量才能正确统计**

**不过，二者也可以分开，详见下面SPFA的处理**

1. BFS：扩展过程满足拓扑序
2. Dijkstra：有效出队序列一定满足拓扑序

在负权图中，只能用floyd/SPFA算法；而SPFA对于每个点的“最短路确定”是不满足拓扑序的——即对于最短路拓扑图来说，SPFA对其中每个点的更新，是不满足拓扑序的；结合上面的解释，由于最短路数量的统计和点的更新都位于松弛操作，此时如果没有正确的递推顺序，则最短路数量统计就无法进行。

修正方法：将“最短路更新”和“最短路数量统计”分开，不再同时在松弛操作处更新

跑完SPFA，最短路拓扑图就构建出来了；然后从头跑这个图，单独处理最短路数量统计
