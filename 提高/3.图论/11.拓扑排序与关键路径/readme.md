### 关键路径
在DAG中，线性时间复杂度求**最长路**

首先，回忆关键路径

求出拓扑序后，用正拓扑序求earliest数组，用逆拓扑序求latest数组

然后遍历所有边，统计关键路径

不过这里我们不取工程意义，我们只关注求earliest和latest数组

正拓扑序求earliest数组，其实就是求从起点出发到达各点的最长路

松弛方程：earliest[j] = max(earliest[j], earliest[i] + w[i][j])

那latest数组呢？其实是反向求从终点出发到达各点的最长路

#### 总结
其实不需要把关键路径单独拿出来考虑，本质都属于DAG的DP问题

对于正向求解，把earliest的max换成min，就变成了求最短路

因此，单纯了解关键路径这个概念，然后用DP视角来看待DAG即可
