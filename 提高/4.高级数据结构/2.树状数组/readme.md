###  操作

> 每个操作时间复杂度为$O(\log n)$
1. 快速求前缀和
2. 修改某一个数

对于用数组去维护前缀和，时间复杂度为$O(n)$；

修改某个数，时间复杂度为$O(1)$

但总时间复杂度取决于最坏的步骤，因此总体为$O(n)$

此时从总体来看，树状数组就要更优秀一些

### 基本原理
基于二进制分解的思想，树状数组的核心是利用二进制的性质来高效地计算前缀和并更新

对于数字$x$，表示成二进制为：$x=2^{i_k}+2^{i_{k-1}}+\cdots+2^{i_1}$，其中$i_k \geq i_{k-1} \geq \cdots \geq i_1$，$k \leq \log x$

下面把上述$x$视为下标来考虑：如果我们想求$a$数组的区间和$[1,x]$，可以用$i$把区间划分为$k$部分：

$(x-2^{i_1},x]$, 区间长度为$2^{i_1}$

$(x-2^{i_1}-2^{i_2},x-2^{i_1}]$, 区间长度为$2^{i_2}$

$\cdots$

$(0,x-2^{i_1}-2^{i_2}-\cdots-2^{i_{k-1}}]$, 区间长度为$2^{i_k}$

如果我们可以把这$\log x$个区间的和都维护好，那么就可以在$O(\log x)$的时间内求出$a[1,x]$的和

总结上式，发现规律：对于区间$(L,R]$，长度一定是$R$的二进制表示的最后一位1的权重

求某个数的最后一个1有一个经典的方法：$lowbit(x)=x \& (-x)$

因此，右边界$R$和区间长度$lowbit(R)$已确定，则上述区间可以改写为：

$[R-lowbit(R)+1,R]$(注意左边通过+1闭上了), 可用$C[R]$来记录对应区间的总和

下面探究所有的$C$之间存在怎样的关系：

$C[x]=sum(a[x-lowbit(x)+1,x])$, $a$是原数组

从$1$开始枚举，写出对应的$C$：

| $x$ | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| $C[x]$ | $a[1]$ | $a[1,2]$ | $a[3]$ | $a[1,4]$ | $a[5]$ | $a[5,6]$ | $a[7]$ | $a[1,8]$ |

| $x$ | 9 | 10 | 11 | 12 | 13 | 14 | 15 | 16 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| $C[x]$ | $a[9]$ | $a[9,10]$ | $a[11]$ | $a[9,12]$ | $a[13]$ | $a[13,14]$ | $a[15]$ | $a[1,16]$ |

在覆盖图中可以看出，16位于第1行，8位于第2行，4和12位于第3行，2、6、10、14位于第4行，1、3、5、7、9、11、13、15位于第5行

第1行区间长度为$2^4$，第2行区间长度为$2^3$，第3行区间长度为$2^2$，第4行区间长度为$2^1$，第5行区间长度为$2^0$

对于第1行的$C[16]$来说，最少可以用哪些部分算出来？这是一个区间覆盖问题，要求不重不漏：

$C[16]=a[1,16]=C[8]+C[12]+C[14]+C[15]+a[16]$

第2行：$C[8]=a[1,8]=C[4]+C[6]+C[7]+a[8]$

第3行：$C[12]=a[9,12]=C[10]+C[11]+a[12]$

这个图看起来其实像一棵树，上面的“分解”其实就是一系列子结点

下面举出一个具体例子：

$x=...100...$, 最后一个1后面有$k$个0，则$C[x]$表示以x为右端点，长度为lowbit(x)——其实就是$2^k$

根据上面的举例，$C[x]$总是包含$a[x]$，因此剩下的区间为$[x-2^k+1,x-1]$

$x-1=...011...$，可以发现后面变成$k$个1了，而这$k$个1就可以按上面所提到的进行区间划分了

这里以$k=4$为例，设$x=10000$(十进制为16)，$x-1=01111$(十进制为15)。令$t=01111$

对这个部分，按照上面$k$个区间划分来进行，其中$i_1=0,i_2=1,i_3=2,i_4=3$

$(01110,01111]\to(t-2^0,t]\to(14,15]\to C[15]$

$(01100,01110]\to(t-2^0-2^1,t-2^0]\to(12,14]\to C[14]$

$(01000,01100]\to(t-2^0-2^1-2^2,t-2^0-2^1]\to(8,12]\to C[12]$

$(00000,01000]\to(t-2^0-2^1-2^2-2^3,t-2^0-2^1-2^2]\to(0,8]\to C[8]$

合起来，就是$C[16]=C[8]+C[12]+C[14]+C[15]+a[16]$

代入数据给出例子后清晰多了，对于$2^{i_k}$的减法，在左边的体现就是抹去一个1

归纳一下：$C[x]=a[x]+C[x-1]+C[x-1-lowbit(x-1)]+...$，直到为0(也就是彻底用二进制表示出x-1)

上面讲了用父结点求子结点，那么如何用子结点来求父结点呢？

这一点对于修改很重要，因为对于某个节点来说，修改了它，所有依赖它的父结点的值都会变化

还是将整个数组看作数——毕竟本身就叫做“树状数组”。用树的视角来看就会很简单了

对于要修改的x，其形式为x=0110...0，则它的直接父亲p为1000......0

则有p=x+lowbit(x)，迭代地求下去直到树根，就能修改所有依赖于x的父结点了。考虑到树结构，显然迭代次数为$O(\log n)$

上述过程可以视为树中某点到根结点的一条路径

```c++
// O(nlogn)初始化树状数组，长度为n，a为原数组
// tr[i]表示以i结尾，长度为lowbit(i)的区间的和(同上方的C[i])
for (int i = 1; i <= n; i++) {
    add(i, a[i]);
}

// O(n)初始化树状数组，根据上述的分析来
// 由于只会用到所有的树边，因此为线性复杂度
// 以16为例，在i从1到15时，tr[8,12,14,15]已经分别累加给tr[16]了；i=16时，tr[16]再加上a[16]，tr[16]就完成了初始化
// 可以认为是一种“自底向上”的线性扫描初始化方法
for (int i = 1; i <= n; i++) {
    tr[i] += a[i]; // 先加上自己原有的值
    int j = i + lowbit(i); // 找到直接父节点
    if (j <= n) {
        tr[j] += tr[i]; // 将当前区间的和整体累加给父亲
    }
}

// 修改
// x为要修改的下标，c为要修改的值
void add(int x, int c) {
    for (int i = x; i <= n; i += lowbit(i)) {
        tr[i] += c;
    }
}

// 查询[1,x]的和
// 注意，用前缀和的思想依然可以求出任意区间的和
// [L,R] = [1,R] - [1,L-1]
int sum(int x) {
    int res = 0;
    for (int i = x; i > 0; i -= lowbit(i)) {
        res += tr[i];
    }
    return res;
}
```

### 扩展
#### 差分
给某一段区间$[L,R]$的每个数加上一个数$c$，即$a[L,R] += c$

显然，使用差分即可。对原数组$a[n]$，求其差分数组$b[n]=a[n]-a[n-1]$

令$b[L] += c, b[R+1] -= c$，然后对b求前缀和，得到修改后的数组a

#### 差分+公式
普通树状数组，实现“单点修改，区间求和”；差分树状数组，实现“区间修改，单点查询”

这里，我们扩展成“区间修改，区间求和”

令tr1维护差分数组b用于求a，tr2维护$i*b[i]$

下面拆解一下tr2的作用：

求$a[1,x]$的区间和，即$a[1]+a[2]+...+a[x]$

即$b[1]+(b[1]+b[2])+(b[1]+b[2]+b[3])+...+(b[1]+b[2]+...+b[x])$

这个式子可以改写为$(b[1]+b[2]+...+b[x])×(x+1)+(1×b[1]+2×b[2]+...+x×b[x])$

根据之前的分析，只要能求出a[1,x]区间和，就能求出任意区间的区间和，不多赘述
