### A*算法——启发式搜索

A*算法解决的问题和双向广搜是一样的

即优化搜索空间过大的BFS，减少不必要的状态扩展

**边权要求：非负即可**

A*算法是通过加一个启发函数来解决这个问题的

启发函数的作用是，从起点开始搜，只需要搜非常少的状态，就可以搜到到达终点的最短路了

因此，A*也不适用于搜索空间比较小的BFS问题

**队列内容：**

从起点到当前点的真实距离a

从当前点到终点的估计距离b

排序字段：a + b

**估计距离b的计算，来源于启发函数**

A*：扩展的时候，选择估计距离b最小的点来扩展

操作：

1. 把BFS中的队列换成优先队列
2. 终点判断写在出队处，以及重复出队判断
3. 遍历所有邻边，入队
4. **一定要保证有解才能用A*算法，否则效率不如BFS**

可以发现，A*形式上很像Dijkstra算法，但二者无关



**A*算法讲解**

设d[s]表示从起点到s的最短距离，g[s]表示从s到终点的最短距离

那么整条路径(从起点到终点且经过s)的真实距离为d[s] + g[s]

设f[s]表示从s到终点的估计距离

若要A*算法成立，则需要保证f[s] <= g[s]

这样才能保证，终点第一次出队时，就是最优答案

因为“真正的最优情况”一定会入队(因为是联通的)，我们必须保证估计的情况比它更早出队才行

**A*算法只能保证终点出队时为最小值，无法保证中间的点出队时为最小值**

**因此，会出队多次；因此出队时不能判重；A*算法只有终点需要“判重”，实际上只需要写终点判断即可**



**如何设计启发函数？**

回到定义，f[s] <= g[s]

最优结果g[s]可能还面临一些“阻碍”，因此f[s]的设计，无视这些阻碍即可

s是当前状态，直接让它“无视地形”，强行走到终点，即可保证一定满足f[s] <= g[s]
