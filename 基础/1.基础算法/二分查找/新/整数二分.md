首先，二分依赖**单调性**

二分的本质是边界问题：

对于[l,r]的整数，设分界点为mid，可以分成两部分：

`[l,mid]和[mid+1,r]` / `[l,mid-1]和[mid,r]`

上面两种划分方式，对应两种模板

注意：整数二分是以整数为基础的，因此**分界点mid会严格属于其中的一部分**

而且，l和r都在闭区间中，属于可以取到的范畴；因此做题时，**将答案能取到的边界直接赋值给l和r即可**

相比旧理解考虑两数之间的分隔作为边界，新理解直接将分界线视为具体元素，更加统一

这两种划分区别在于什么呢？

先给结论：mid在哪，哪就是我们需要的一边

注意，代码一开始我们并不知道mid在哪，只是不断地二分并更新mid，直到它被锁定下来

因此，**二分的最终目的，是找到mid**

设数组是**单调递增**的

对于`[l,mid][mid+1,r]`，mid在左边，那么[l,mid]满足题设条件，mid为**满足条件的最大值**

对于`[l,mid-1][mid,r]`，mid在右边，那么[mid,r]满足题设条件，mid为**满足条件的最小值**



##### 代码模拟1
满足题意的部分在左边——对应`[l,k][k+1,r]`划分
1. `mid=l+r**+1**>>1`，注意这里的mid是不断更新的状态，而上方的k是最终锁定的值。+1的原因是：若不+1，由于除法是下取整，若l=r-1(l和r只差1)，则mid=l；若下面进入check后返回true，则l=mid=l，下次二分区间仍然是`[l,r]`，死循环；而+1后，mid=r，下次二分区间为`[r,r]`，锁定mid=r=k，结束循环。总之，+1的目的是为了避免死循环，保证mid能够向右移动
2. `if (check(mid))`，查询计算出的mid值是否应当在目标边界，即左半边
3. 若true，则此时`mid∈[l,k]`，处于有解范围中，应当让mid更逼近右边界；但不确定此时mid是否已经位于右边界，因此更新`l=mid`。下一次二分对应的区间为[mid,r]，同时`mid<=k`，因此解k依然处于二分区间
4. 若false，则此时`mid∈[k+1,r]`，处于无解范围中，因此要直接让mid向左，更新`r=mid-1`，则下一次二分区间为`[l,mid-1]`。因为上次已经确定了mid不满足条件，所以下一次二分的右边界，应当是其往左一个位置，mid-1(r=mid-1的原因)
5. 最后l和r最终会锁定在k上，返回l或r都行；还可以根据题目需求任意调整返回值，如分界点k的右边l+1
false的二分区间为[l,mid-1]，true的二分区间为[mid,r]。注意与k划分区别

##### 代码模板1
```c++
int bsearch1(int l, int r) {
    while (l < r) {
        int mid = l + r + 1 >> 1;
        if (check(mid)) {
            l = mid;
        } else {
            r = mid - 1;
        }
    }
    return l;
}
```

##### 代码模拟2
满足题意的部分在右边——对应`[l,k-1][k,r]`划分
1. `mid=l+r>>1`，无需再`+1`
2. `if (check(mid))`，查询计算出的mid值是否应当在目标边界，即右半边
3. 若true，则此时`mid∈[k,r]`，处于有解范围中，应当让mid更逼近左边界；但不确定此时mid是否已经位于左边界，因此更新`r=mid`。下一次二分对应的区间为`[l,mid]`，同时`mid>=k`，因此解k仍然处于二分区间
4. 若false，则此时`mid∈[l,k-1]`，处于无解范围中，因此要直接让mid向右，更新`l=mid+1`，则下一次二分区间为`[mid+1,r]`。因为上次已经确定了mid不满足条件，所以下一次二分的左边界，应当是其往右一个位置，mid+1(l=mid+1的原因)
5. 最后l和r最终会锁定在k上，返回l或r都行；还可以根据题目需求任意调整返回值，如分界点k的左边l-1
true的二分区间为[l,mid]，false的二分区间为[mid+1,r]。注意与k划分区别

##### 代码模板2
```c++
int bsearch1(int l, int r) {
    while (l < r) {
        int mid = l + r >> 1;
        if (check(mid)) {
            r = mid;
        } else {
            l = mid + 1;
        }
    }
    return l;
}
```

巧记：若答案在左，则l=mid(T), r=mid-1(F)；若答案在右，则r=mid(T), l=mid+1(F)

答案在左需+1，真左不变，假右更左；答案在右，真右不变，假左更右
